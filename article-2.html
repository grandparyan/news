什麼是 ip addr？一篇搞懂如何查看 Linux 和 macOS 網路IP位址

發布日期：2025-11-14 | 分類：Linux 教學

當您使用 Linux 伺服器或桌面系統時，如果需要查詢網路設定，`ip addr` 是現代 Linux 發行版中最重要、最標準的指令。

`ip addr` 是 `iproute2` 工具包的一部分，它取代了傳統（且現已過時）的 `ifconfig` 指令。`ip addr` 提供更詳細、更準確的網路介面資訊。

這裡需要釐清一個常見的觀念：`ip addr` 是 Linux 系統常用的指令。

Apple (macOS) 系統，以及其他 BSD 系統，傳統上使用的也是 ip addr 指令。而 Windows 系統則是使用 ipconfig。雖然它們的名稱很相似，但 ip addr 因其功能的強大與現代化，已成為 Linux 上的新標準。

什麼是 ip addr？

`ip addr` (address) 是一個命令列工具，用於顯示和管理 Linux 系統上的網路介面卡及其 IP 位址設定。

如何使用 ip addr？

使用 `ip addr` 非常簡單。您只需要打開「終端機」(Terminal)，然後輸入： 

ip addr

您也可以使用更短的別名，例如 `ip a`，它們的效果完全相同。 

ip a

您該看哪些重要資訊？

當您執行 `ip addr` 後，會看到類似以下的輸出： 

1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
       valid_lft forever preferred_lft forever
    inet6 ::1/128 scope host 
       valid_lft forever preferred_lft forever
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000
    link/ether 00:1a:2b:3c:4d:5e brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 brd 192.168.1.255 scope global dynamic noprefixroute eth0
       valid_lft 86032sec preferred_lft 86032sec
    inet6 fe80::21a:2bff:fe3c:4d5e/64 scope link noprefixroute 
       valid_lft forever preferred_lft forever

雖然資訊看起來很多，但您只需要關注幾個關鍵點（以上述 `eth0` 介面為例）：

  • `eth0`： 這是網路介面卡的名稱。`eth` 通常代表乙太網路（有線網路），`wlan` 或 `wlp` 則代表無線網路。
  • `inet 192.168.1.100/24`： 這就是您電腦的 **IPv4 位址** (`192.168.1.100`)。後面的 `/24` 是 CIDR 標記法，代表子網路遮罩 (`255.255.255.0`)。
  • `link/ether 00:1a:2b:3c:4d:5e`： 這是您的網路卡**硬體位址 (MAC Address)**。
  • `inet6 fe80::...`： 這是您的 **IPv6 位址**。

如何查看路由表 (Gateway)？

`ip addr` 專注於顯示「位址」。如果您想查看「預設閘道」(Default Gateway)，您需要使用 `iproute2` 工具包中的另一個指令： 

ip route

執行後，您會看到一行 `default via 192.168.1.1 ...`，那個 `192.168.1.1` 就是您的預設閘道。
