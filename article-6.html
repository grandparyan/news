網路協定資料存哪裡？一篇搞懂 Cookie, 金鑰, 快取

發布日期：2025-11-18 | 分類：網路安全

不同的網路協定，出於不同的目的被設計，因此它們儲存資料的方式和位置也大不相同。有些協定（如 Telnet）幾乎不儲存任何資料，而另一些（如 HTTP）則嚴重依賴儲存在您電腦上的資料。

了解這些資料（例如 Cookie、金鑰、快取）儲存在哪裡，是網路安全和隱私保護的關鍵第一步。

HTTP (Port 80) / HTTPS (Port 443)

這是您我每天用來瀏覽網頁的協定。由於 HTTP 本身是「無狀態的」(stateless)，網站需要一種方式來「記住」您是誰。這就催生了儲存在瀏覽器中的各種資料。

Cookie (最重要的資料)

Cookie 是網站儲存在您電腦上的一小段文字。它就像是網站發給您的「識別證」。

  • 功用： 用於儲存登入狀態（「保持登入」）、購物車內容、個人化設定、以及追蹤您的瀏覽行為（廣告商的最愛）。
  • 存取位置： 儲存在您的**網頁瀏覽器**中。
  • 如何查看： 在瀏覽器中按下 `F12` 打開開發者工具，切換到「Application」(應用程式) 標籤頁，您就可以在左側選單中找到 `Cookies`。

LocalStorage 和 SessionStorage

這兩種是較現代的網頁儲存技術，允許網站在您的瀏覽器中儲存比 Cookie 更多的資料。

  • 功用： `LocalStorage` 用於永久儲存資料（例如網頁應用的設定）；`SessionStorage` 則在您關閉分頁後就會自動清除。
  • 存取位置： 同樣在瀏覽器「Application」標籤頁中。

瀏覽器快取 (Cache)

瀏覽器會將網站的靜態檔案（如圖片、CSS、JavaScript 檔案）下載並儲存在您的電腦硬碟中。

  • 功用： 當您下次再訪問同一個網站時，瀏覽器會直接從硬碟讀取這些檔案，而不是重新下載，藉此大幅加快網頁載入速度。

SSH (Port 22)

SSH 是一種安全的遠端登入協定。它的資料儲存重點完全不在於「追蹤」，而在於「驗證」，確保您是您，且您連線的伺服器是您要找的伺服器。

Known Hosts (known_hosts)

這是 SSH 最重要的安全機制之一。

  • 功用： 當您第一次連線到一台 SSH 伺服器時，伺服器會傳送它的「公鑰指紋」。您的 SSH 客戶端會詢問您是否信任它。一旦您回答「yes」，這個指紋就會被儲存起來。
  • 存取位置 (Linux/macOS)： ~/.ssh/known_hosts
  • 安全機制： 下次您再連線時，客戶端會核對伺服器傳來的指紋是否與 `known_hosts` 中儲存的一致。如果不一致（可能是伺服器重灌，或更糟的——您正遭受中間人攻擊），SSH 會立刻中斷連線並發出嚴重警告。

SSH 金鑰 (id_rsa / id_rsa.pub)

這是用來取代密碼登入的、更安全的方式。

  • 功用： `id_rsa` (私鑰) 儲存在您的電腦上，絕對不能外洩；`id_rsa.pub` (公鑰) 則放在您要登入的遠端伺服器上。
  • 存取位置 (Linux/macOS)： ~/.ssh/

RDP (Port 3389)

RDP 是 Windows 遠端桌面協定，它也會在您的電腦上儲存一些快取資料以提升效能。

連線設定 (.rdp 檔案)

  • 功用： RDP 客戶端會儲存您連線過的 IP 位址、使用者名稱和顯示設定。
  • 存取位置： 儲存在 `.rdp` 檔案中，或 Windows 登錄檔 (Registry) 裡。

點陣圖快取 (Bitmap Cache)

  • 功用： 類似瀏覽器快取。RDP 客戶端會將遠端桌面上不常變動的區域（例如桌布、圖示）以圖片（點陣圖）的方式儲存在本機硬碟。
  • 存取位置 (Windows)： %LOCALAPPDATA%\Microsoft\Terminal Server Client\Cache

儲存的認證 (Credentials)

  • 功用： 如果您在連線時勾選了「記住我的認證」，您的 RDP 密碼會被加密儲存在 Windows 的「認證管理員」中。

Telnet (Port 23)

Telnet 是一種極度不安全的明文協定。

  • 功用： 遠端命令列存取。
  • 存取位置： **無。** Telnet 協定本身是完全無狀態的，它**不會在您的電腦上儲存任何資料**（沒有 Cookie、沒有金鑰、沒有快取）。
  • 資料在哪裡？ Telnet 的所有資料（包括您輸入的帳號密碼）都在**網路上以明文傳輸**。這就是為什麼它很危險：攻擊者不需要存取您的電腦，只需要在網路上「竊聽」(Sniffing)，就能擷取您的所有資訊。
