什麼是網路服務(Port)？一篇搞懂 Telnet, SSH, HTTP, RDP

發布日期：2025-11-16 | 分類：網路基礎

在上一篇 Nmap 教學中，我們提到了「連接埠掃描 (Port Scanning)」。但什麼是連接埠 (Port)？它又和網路服務 (Service) 有什麼關係？

您可以把電腦的 IP 位址（例如 `192.168.1.100`）想像成一棟公寓大樓的地址。而**連接埠 (Port)** 就是這棟大樓裡**每個房間的房號**。

每當電腦需要提供一項服務（例如網站服務），它就會在特定的一個「房號」上等待。當您的瀏覽器要訪問這個網站時，它不僅需要知道大樓地址 (IP)，還需要知道房號 (Port)，這樣才能正確找到服務。

以下我們將介紹幾個最常見的網路服務，以及它們預設使用的「房號」(Port)。

服務名稱 (Protocol) 協定類型 (Protocol) 預設埠號 (Default Port) 備註 (Note)
Telnet TCP 23 傳輸明文資料，極不安全。
SSH (Secure Shell) TCP 22 安全的遠端登入與指令執行。
HTTP TCP 80 網站傳輸協定，未加密。安全替代方案為 HTTPS (443)。
FTP (File Transfer) TCP 21 傳輸明文
NFS (Network File System) UDP / TCP 2049 舊版通常為 UDP/2049加密較為簡易
SMB (Server Message Block) TCP 445 用於 Windows 檔案和印表機共享。在較舊系統中可能仍使用 NetBIOS over TCP/IP (Port 137, 138, 139)。
RDP (Remote Desktop Protocol) TCP 3389 遠端桌面連線。

Telnet (Port 23)

Telnet 是一種非常古老的遠端連線協定。它允許您從一台電腦登入到另一台遠端的主機，並像在本地一樣操作它的命令列（文字介面）。

⚠️ 安全警告： Telnet 最大的問題是它所有的傳輸內容，包括您輸入的帳號和密碼，都是**「明文傳輸」**（沒有加密）。這意味著網路上的任何人都可以輕易竊聽到您的登入資訊。因此，Telnet 現在幾乎已被完全淘汰，**絕對不應該**在公共網路上使用。

SSH (Port 22)

SSH (Secure Shell) 是 Telnet 的現代繼承者。它的功能與 Telnet 相同——提供遠端命令列存取——但它解決了 Telnet 最大的安全缺陷。

SSH 會在您的電腦和遠端主機之間建立一個**「加密通道」**。所有傳輸的資料（包括密碼）都會被加密，確保駭客無法竊聽。SSH 是目前管理 Linux/macOS 伺服器的標準作法。

HTTP (Port 80)

HTTP (Hypertext Transfer Protocol) 是您我每天都在使用的協定。當您在瀏覽器網址列輸入一個 `http://` 開頭的網址時，您的瀏覽器就是透過 80 Port 向網站伺服器請求網頁資料。

(您在列表中提到了兩次 http，我猜其中一個是指 `HTTPS`，這更為重要。)

HTTPS (Port 443)

HTTPS (HTTP Secure) 是 HTTP 的安全版本。就像 SSH 保護 Telnet 一樣，HTTPS 使用加密（通常是 SSL/TLS 憑證）來保護您和網站之間的通訊。

當您在網站上輸入密碼、信用卡號或個人資料時，HTTPS 可以確保這些資訊不會在傳輸過程中被竊取。這就是為什麼現代瀏覽器會將使用 Port 80 (HTTP) 的網站標記為「不安全」，並優先使用 Port 443 (HTTPS)。

RDP (Port 3389)

RDP (Remote Desktop Protocol) 是由微軟開發的協定。與 SSH 或 Telnet 只傳輸文字不同，RDP 允許您遠端存取一台 Windows 電腦的**完整圖形桌面**。

當您使用 Windows 內建的「遠端桌面連線」功能時，您就是在透過 3389 Port 連線到另一台電腦，操作它的滑鼠和鍵盤，就像您親自坐在那台電腦前面一樣。

FTP (Port 21)

FTP (File Transfer Protocol) 是一種專門用來在網路上傳輸檔案的古老協定。Port 21 用於建立連線和傳輸指令，而資料傳輸本身通常會使用另一個 Port (Port 20 或動態 Port)。

⚠️ 安全警告： 類似 Telnet，標準的 FTP 也是以**明文**傳輸帳號、密碼和檔案內容，非常不安全。現在強烈建議使用其安全版本，如 `SFTP` (基於 SSH) 或 `FTPS` (基於 SSL/TLS)。

NFS (Port 2049)

NFS (Network File System) 主要用於 Unix 和 Linux 系統之間，它允許一台電腦上的使用者像存取本地檔案一樣，存取網路上另一台電腦的檔案系統。

舊版的 NFS 主要使用 UDP，而新版本 (NFSv4) 則強烈建議並主要使用 TCP，以提供更可靠的連線。

SMB (Port 445)

SMB (Server Message Block) 是 Windows 系統檔案共享的核心協定。當您在 Windows「網路上的芳鄰」中存取另一台電腦的共享資料夾或印表機時，您就是在使用 SMB。

在現代 Windows 系統中，SMB 直接運作在 TCP 445 Port 上。在非常老舊的系統 (Windows 2000 之前)，它可能還會依賴 NetBIOS (Port 137-139)。
